Builds

What I've built

Real workflows, tested live on my own n8n instance. "Production" means every path is confirmed working. "Portfolio" means it runs but is built for demonstration, not a live client.

Client intake & billing pipeline

Production

Built for an accounting firm. A client submits a form - 90 seconds later a VAT-inclusive quote is in their inbox. Detects English vs Bisaya/Cebuano, routes differently, drafts an AI reply, calculates 12% VAT with a BIR compliance note, and logs everything to Google Sheets.

The Bisaya detection kept misclassifying. The fix was a keyword fallback layer - so even when the model gets it wrong, the workflow doesn't.

18 nodes 6 paths tested Ollama · Sheets · Resend

Invoice & tax auditor

Production

Watches Gmail for PDF invoices. Extracts five fields using a local AI model - freelancer name, invoice number, amount, currency, country. Fetches live exchange rates, applies Philippine withholding tax logic (PH: 10%, international with waiver: 0%, without: 25%), logs to Baserow, and generates a styled payment voucher.

Zero manual data entry. The model had to extract from unstructured PDFs across multiple invoice formats - tested with Chrome-printed PDFs only (Microsoft Print to PDF produces image-only files with no extractable text).

16 nodes PH + international tax Ollama · Baserow · Resend

Gmail → Ollama → draft reply

Production

Monitors Gmail for emails tagged with a label. Passes each one to a local Ollama model to generate a professional reply. Saves the draft back to the correct thread in Gmail Drafts - ready to review and send.

Label-based trigger, not keyword filter - no false positives. Draft saves to the original thread via threadId so replies never land in the wrong conversation.

5 nodes Ollama · Gmail API

Lead follow-up pipeline

Draft

Watches Gmail for leads. Filters bots and self-sent emails. Scores each lead 1-10, classifies intent, flags urgency, writes a one-sentence summary. A draft reply lands in Gmail Drafts before you've touched your inbox. High-urgency leads get an alert email too.

n8n's native "Remove Label" node returns "Success" on every run but never removes the label. The same emails kept re-triggering. Replaced with a direct Gmail API call via HTTP Request node.

17 nodes AI scoring + urgency routing Ollama · Gmail API · Baserow

Inventory reconciliation engine

Portfolio

Reconciles inventory across four sources: Postgres ERP, Shopify, 3PL US, 3PL EU, and 3PL Asia via SFTP. Detects discrepancies, sends a Slack alert, pauses for human approval, then pushes corrections to Shopify via GraphQL mutations. SFTP sub-workflow retries three times before alerting DevOps.

Demo version available - all external calls replaced with simulated data so it runs live without credentials. Same 25-node structure as the production build.

43 nodes total Human approval gate Postgres · Shopify · Slack · SFTP

Supply chain & financial compliance

Draft

Processes purchase orders from Gmail attachments - PDF or Excel. Extracts line items with AI, checks stock in Google Sheets, deducts inventory per SKU in batches, calculates 12% VAT and zone-based shipping, generates an HTML invoice, and routes partial/stockout orders to a separate procurement alert.

31 nodes across two branches. Inventory check fetches the full sheet once and does all SKU lookups in JavaScript - avoids a per-SKU API loop that would spike CPU on a local instance.

31 nodes PDF + Excel input Ollama · Sheets · Resend
